# 專案管理 Project Management

> 確保專案按時、按質、按預算完成

## 適用場景

- 專案計畫制定與排程
- 敏捷/Scrum 流程規劃
- 風險識別與管理
- 團隊協作與溝通
- 專案進度追蹤

## 敏捷/Scrum 框架

### Scrum 角色

| 角色 | 職責 |
|------|------|
| **Product Owner** | 定義需求優先級、維護 Product Backlog |
| **Scrum Master** | 促進流程、移除障礙、保護團隊 |
| **Development Team** | 交付增量、自組織、跨職能 |

### Scrum 事件

```
Sprint（2-4 週）
│
├── Sprint Planning（4-8 小時）
│   ├─ What: 這個 Sprint 要完成什麼？
│   └─ How: 如何完成？
│
├── Daily Standup（15 分鐘）
│   ├─ 昨天做了什麼？
│   ├─ 今天要做什麼？
│   └─ 有什麼障礙？
│
├── Sprint Review（2-4 小時）
│   └─ Demo 完成的功能給 stakeholders
│
└── Sprint Retrospective（1.5-3 小時）
    ├─ 什麼做得好？
    ├─ 什麼可以改進？
    └─ 下個 Sprint 要嘗試什麼？
```

### User Story 格式

```markdown
## 標準格式
As a [角色]
I want [功能]
So that [價值]

## Story Points 估算（Fibonacci）
1 - 非常小，幾小時內完成
2 - 小，半天到一天
3 - 中等，1-2 天
5 - 較大，2-3 天
8 - 大，接近一週
13 - 非常大，考慮拆分
```

## 風險管理

### 風險矩陣

```
影響
  ↑
高 │ 3 │ 6 │ 9 │  ← 高風險區（紅色）
  │───┼───┼───│
中 │ 2 │ 4 │ 6 │  ← 中風險區（黃色）
  │───┼───┼───│
低 │ 1 │ 2 │ 3 │  ← 低風險區（綠色）
  └───┴───┴───┘→
   低  中  高  機率
```

### 風險應對策略

| 策略 | 說明 | 適用情況 |
|------|------|----------|
| **規避** | 改變計畫消除風險 | 高影響、可避免 |
| **轉移** | 轉移給第三方 | 專業領域風險 |
| **緩解** | 降低機率或影響 | 無法完全避免 |
| **接受** | 準備應變計畫 | 低影響或無法處理 |

## 估算技巧

### Planning Poker

```
1. 講解 User Story
2. 團隊成員各自估算
3. 同時亮牌
4. 討論差異最大的
5. 重新估算直到達成共識
```

### 三點估算

```
預期時間 = (樂觀 + 4×最可能 + 悲觀) / 6

範例：
- 樂觀：3 天
- 最可能：5 天
- 悲觀：10 天
- 預期 = (3 + 4×5 + 10) / 6 = 5.5 天
```

## 常見錯誤

| 錯誤 | 正確做法 |
|------|----------|
| 範圍蔓延不管 | 走變更流程，評估影響 |
| 進度報喜不報憂 | 及早暴露問題，尋求幫助 |
| 估算過度樂觀 | 加入緩衝，使用歷史數據 |
| 忽略依賴關係 | 識別並管理依賴項 |
