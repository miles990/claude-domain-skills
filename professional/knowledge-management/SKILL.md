# 知識管理 Knowledge Management

> 建立個人知識系統，讓資訊成為可複利的資產

## 適用場景

- 建立個人筆記系統
- 設計資訊整理工作流
- 打造第二大腦
- 數位工具選擇與整合
- 知識輸出與分享


## PARA 分類系統

```
┌─────────────────────────────────────────────────────────────────┐
│  PARA 資料夾結構                                                │
│                                                                 │
│  📁 Projects (專案)                                             │
│  │   └─ 有明確截止日期的任務                                    │
│  │       例：產品發布、報告撰寫、活動籌備                       │
│  │                                                              │
│  📁 Areas (領域)                                                │
│  │   └─ 持續維護的責任範圍                                      │
│  │       例：健康、財務、職涯、家庭                             │
│  │                                                              │
│  📁 Resources (資源)                                            │
│  │   └─ 感興趣的主題參考資料                                    │
│  │       例：投資、寫作、AI、旅遊                               │
│  │                                                              │
│  📁 Archives (封存)                                             │
│      └─ 已完成或不再活躍的項目                                  │
│                                                                 │
│  分類原則：按「可操作性」而非「主題」                           │
└─────────────────────────────────────────────────────────────────┘
```

## 筆記方法

### 卡片盒筆記法 (Zettelkasten)

```markdown
## 原則

1. **原子性** - 一張卡片一個概念
2. **連結性** - 卡片之間互相連結
3. **自己的話** - 用自己的理解重寫
4. **永久筆記** - 經過思考的筆記

## 筆記類型

| 類型 | 說明 | 保留時間 |
|------|------|----------|
| 閃念筆記 | 快速捕捉想法 | 臨時 |
| 文獻筆記 | 閱讀時的摘錄 | 參考用 |
| 永久筆記 | 自己消化後的理解 | 永久 |
| 專案筆記 | 特定專案相關 | 專案期間 |

## 筆記模板

標題：[概念名稱]
日期：YYYY-MM-DD
來源：[出處]
連結：[[相關筆記1]] [[相關筆記2]]

---
[用自己的話解釋這個概念]

---
為什麼重要：
可以應用在：
```

### 漸進式摘要

```
┌─────────────────────────────────────────────────────────────────┐
│  漸進式摘要 (Progressive Summarization)                         │
│                                                                 │
│  Layer 0: 原始內容                                              │
│  ↓                                                              │
│  Layer 1: 粗體標記重點句                                        │
│  ↓                                                              │
│  Layer 2: 螢光標記關鍵詞                                        │
│  ↓                                                              │
│  Layer 3: 頂部摘要（幾句話）                                    │
│  ↓                                                              │
│  Layer 4: 原創見解和應用                                        │
│                                                                 │
│  只在需要時才進行下一層，按需深入                               │
└─────────────────────────────────────────────────────────────────┘
```

## 數位工具流

### 工具選擇矩陣

| 需求 | 推薦工具 | 特點 |
|------|----------|------|
| **雙向連結筆記** | Obsidian, Logseq | 本地、Markdown |
| **全功能 All-in-One** | Notion | 雲端、團隊協作 |
| **快速捕捉** | Apple Notes, Drafts | 速度優先 |
| **閱讀稍後** | Readwise, Instapaper | 標註同步 |
| **任務管理** | Todoist, Things | GTD 流程 |
| **長期儲存** | DEVONthink, Evernote | 搜尋強大 |

### 工具流設計

```markdown
## 資訊收集流

瀏覽器 → 稍後閱讀 → 標註 → 筆記軟體 → 永久筆記
         Instapaper   Readwise   Obsidian

## 任務流

收件匣 → 處理 → 專案/待辦 → 執行 → 歸檔
Inbox    Process   Project    Do     Archive

## 每日流程

1. 早上：檢視今日任務
2. 工作：專注執行
3. 隨時：快速捕捉想法
4. 晚上：整理收件匣，規劃明天
```

## 輸入與輸出平衡

```
┌─────────────────────────────────────────────────────────────────┐
│  知識複利公式                                                   │
│                                                                 │
│  輸入 (Input)                                                   │

│  • 課程                                                         │

│  • 對話                     →   │  知識處理   │   →  輸出      │

│                                 └─────────────┘                │
│                                                                 │
│  輸出 (Output)                                                  │
│  • 寫作                                                         │
│  • 教學                                                         │
│  • 創作                                                         │
│  • 應用                                                         │
│                                                                 │
│  關鍵：輸出倒逼輸入，學以致用                                   │
│                                                                 │
│  建議比例：輸入 : 處理 : 輸出 = 3 : 2 : 5                      │
└─────────────────────────────────────────────────────────────────┘
```

## 常見陷阱

| 陷阱 | 解決 |
|------|------|
| 收集狂 | 只收集會用到的 |
| 工具控 | 工具服務目的，不是目的 |
| 過度分類 | 搜尋比分類更重要 |
| 只輸入不輸出 | 強制輸出，學以致用 |
| 追求完美系統 | 開始比完美重要 |

## 進階主題

### 數位花園

```markdown
## 公開學習 (Learning in Public)

原則：
1. 分享學習過程，不只是結果
2. 容許不完美，持續迭代
3. 建立連結，形成網路

形式：
- 部落格 (長文)
- Twitter/X (短文)
- Newsletter (定期)
- 數位花園 (持續更新)
```
