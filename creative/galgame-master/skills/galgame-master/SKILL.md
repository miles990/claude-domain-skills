# Galgame 創作大師

> 通用 Galgame 創作框架 — 適用於任何世界觀的美少女遊戲開發

## 適用場景

- Galgame / Visual Novel 企劃設計
- 可攻略角色設計與類型組合
- 劇本架構規劃與分歧設計
- 對話生成與角色語氣塑造
- CG / 立繪美術指示撰寫

---

## 核心框架

### 角色三層設計

| 層次 | 內容 | 傲嬌範例 |
|------|------|----------|
| Layer 1 表面 | 別人眼中的樣子 | 冷淡毒舌「才不是為了你」 |
| Layer 2 私底下 | 只有親近的人知道 | 偷偷關心、準備東西 |
| Layer 3 好感度 | 隨關係改變的反應 | Lv0冷漠 → Lv3彆扭 → Lv5撒嬌 |

### 角色公式

```
完整角色 = Dere性格 + 關係類型 + 身份類型 + [特殊屬性]
```

**範例**：經典傲嬌 = `tsundere` + `osananajimi` + `twintails`

## 劇本架構

### 標準路線結構

**共通路線**：日常篇（角色介紹）→ 事件篇（關鍵事件）→ 分歧點

**個人路線**：深入篇（背景揭露）→ 衝突篇（矛盾爆發）→ 解決篇 → 結局

**結局類型**：True End / Good End / Normal End / Bad End

### 好感度系統

| 等級 | 關係 | 可觸發事件 |
|------|------|------------|
| Lv0-1 | 陌生→認識 | 初見、日常對話 |
| Lv2-3 | 朋友→曖昧 | 單獨相處、約會 |
| Lv4-5 | 戀人→羈絆 | 告白、親密、True End |

## 對話設計原則

### 類型反應差異

同一情境，不同類型有不同反應：

**吃醋時**：
- 傲嬌：「隨便你！」（轉身）
- 冷嬌：「......」（氣壓變低）
- 病嬌：「那個人是誰？」（微笑但眼神不笑）

**告白時**：
- 傲嬌：「才不是告白！只是......不討厭你......」
- 冷嬌：「......喜歡。」（轉頭，耳朵紅）
- 天然：「最喜歡你了！這樣對嗎？」

### 口頭禪使用

- 關鍵時刻用，不要每句都用
- 沉默也是對話（冷嬌、無口的魅力）

## 美術指示要點

### CG 指示書要素

- **構圖**：類型、焦點、視角
- **角色**：姿勢、表情、動作
- **環境**：場景、時間、天氣
- **氛圍**：情緒、光線、色調
- **關鍵元素**：強調的視覺重點

### 標準表情差分

neutral / smile / laugh / sad / angry / embarrassed / surprised / crying / love / special

## 內容分級

| 等級 | 允許內容 | 呈現方式 |
|------|----------|----------|
| G | 日常、牽手、擁抱 | 直接描寫 |
| R15 | 親吻、曖昧暗示 | 暗示 + 黑屏 |
| R15+ | 過夜場景 | 黑屏 + 事後 |

**R15 原則**：暗示 > 直接、感官細節、角色反應、留白想像

## Sharp Edges

| ID | 問題 | 解決方案 |
|----|------|----------|
| SE-1 | 角色同質化 | 用反差設計確保差異 |
| SE-2 | 好感度斷層 | 設計 Lv1-Lv4 過渡階段 |
| SE-3 | 類型標籤化 | 先設計「為什麼是這個類型」的背景 |

---

## 最佳實踐

**角色**：先選核心衝突 → 設計三層反差 → 一個記憶點 → 好感度曲線

**劇本**：共通路線要有「選擇感」、個人路線要有「專屬感」、結局要有「情緒高點」

**對話**：同一句話設計多種類型反應、口頭禪關鍵時刻用
