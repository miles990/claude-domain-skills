## 核心知識

### 角色設計三層原則

> **「好的 Galgame 角色要有三層」**

```
┌─────────────────────────────────────────────────────────────────┐
│  角色三層設計                                                    │
│                                                                 │
│  Layer 1: 表面印象     →  別人眼中的樣子                        │
│  Layer 2: 私底下       →  只有親近的人知道                      │
│  Layer 3: 好感度變化   →  隨關係改變的反應                      │
│                                                                 │
│  例：傲嬌角色                                                   │
│  表面：冷淡毒舌「才不是為了你」                                 │
│  私底下：偷偷關心、準備東西                                     │
│  好感度變化：Lv0冷漠 → Lv3彆扭關心 → Lv5坦率撒嬌               │
└─────────────────────────────────────────────────────────────────┘
```

### 角色公式

```
完整角色 = Dere性格 + 關係類型 + 身份類型 + [特殊屬性]
```

**範例組合**：
- 經典傲嬌 = `tsundere` + `osananajimi` + `twintails`
- 冰山學姐 = `kuudere` + `senpai` + `kaichou` + `kurokami`
- 小惡魔學妹 = `koakuma` + `kouhai` + `idol`

## 劇本架構系統

### 標準路線結構

```
共通路線（序章 ~ 中盤）
├── 日常篇：角色介紹、關係建立
├── 事件篇：觸發各角色關鍵事件
└── 分歧點：選擇進入哪條個人路線

個人路線（中盤 ~ 結局）
├── 深入篇：角色背景、內心掙扎
├── 衝突篇：核心矛盾爆發
├── 解決篇：一同克服困難
└── 結局：
    ├── True End（最佳結局）
    ├── Good End（好結局）
    ├── Normal End（普通結局）
    └── Bad End（壞結局）
```

### 好感度系統

| 等級 | 數值 | 關係狀態 | 可觸發事件 |
|------|------|----------|------------|
| Lv0 | 0-19 | 陌生人 | 初次見面 |
| Lv1 | 20-39 | 認識 | 日常對話 |
| Lv2 | 40-59 | 朋友 | 單獨相處 |
| Lv3 | 60-79 | 曖昧 | 約會事件 |
| Lv4 | 80-99 | 戀人 | 告白、親密事件 |
| Lv5 | 100 | 羈絆 | 最終事件、True End |

## 對話生成系統

### 情境對話模板

**吃醋場景**：
```
[傲嬌] 「你和那個人很好嘛......隨便你！」（轉身）
[冷嬌] 「......那個人。」（沉默，但氣壓變低）
[天然] 「......你和她感情很好呢。」（不懂自己為何難過）
[病嬌] 「那個人是誰？」（微笑但眼神不笑）
```

**告白場景**：
```
[傲嬌] 「才、才不是告白！只是......不討厭你而已......笨蛋」
[冷嬌] 「......喜歡。」（說完轉頭，耳朵紅透）
[天然] 「我想了很久喔......最喜歡你了！這樣對嗎？」
[元氣] 「我喜歡你！......欸，我說出來了？」
```

## 內容分級系統

| 等級 | 代碼 | 允許內容 | 呈現方式 |
|------|------|----------|----------|
| 全年齡 | G | 日常、牽手、擁抱 | 直接描寫 |
| R15 | R15 | 親吻、曖昧暗示 | 暗示 + 黑屏 |
| R15+ | R15_MAX | 過夜場景 | 黑屏 + 事後 |
| R18 | R18 | 成人內容 | 依平台規範 |

### R15 場景原則

**允許**：
- ✓ 暗示 > 直接描寫
- ✓ 感官細節（觸感、氣味、溫度）
- ✓ 角色反應（臉紅、迴避）
- ✓ 留白讓玩家想像
- ✓ 黑屏 + 事後

**禁止**：
- ✗ 明確的性描寫
- ✗ 裸露超過泳裝程度
- ✗ 強迫性場景

## 最佳實踐

### 角色設計

1. **先選核心衝突**：角色最大的內心掙扎是什麼？
2. **設計三層反差**：表面、私底下、好感度變化
3. **一個記憶點**：讓玩家記住的獨特元素（口頭禪、動作、視覺）
4. **好感度曲線**：Lv0-Lv5 的變化要有明確節奏

### 劇本設計

1. **共通路線要有「選擇感」**：讓玩家覺得自己在影響故事
2. **個人路線要有「專屬感」**：只有這條路線才能看到的內容
3. **結局要有「情緒高點」**：True End 必須讓人感動

### 對話設計

1. **同一句話，不同類型**：給同樣情境設計多種類型的反應
2. **口頭禪要自然**：不要每句都用，關鍵時刻用效果更好
3. **沉默也是對話**：冷嬌、無口的魅力在於「沒說出口的話」

## 常見錯誤

| 錯誤 | 正確做法 |
|------|----------|
| ❌ 角色只有一面 | ✅ 設計三層：表面、私底下、好感度變化 |
| ❌ 好感度沒有過渡 | ✅ Lv0→Lv5 要有明確的變化過程 |
| ❌ 所有角色反應一樣 | ✅ 同一情境不同類型有不同反應 |
| ❌ 口頭禪用太多 | ✅ 關鍵時刻用，平時自然說話 |
| ❌ 類型混亂組合 | ✅ 參考組合禁忌，避免衝突屬性 |
| ❌ 劇情沒有專屬感 | ✅ 每條路線要有獨特的發展和衝突 |

## Sharp Edges

### SE-1: 角色同質化

- **嚴重度**: high
- **情境**: 設計多位可攻略角色時
- **原因**: 只用一種類型，或沒設計足夠反差
- **症狀**: 玩家分不清角色、缺乏攻略動力
- **解決**: 用反差設計（主動型 ↔ 被動型、姐姐型 ↔ 妹妹型）確保角色差異

### SE-2: 好感度斷層

- **嚴重度**: medium
- **情境**: 撰寫不同好感度階段的對話
- **原因**: 直接從 Lv0 跳到 Lv5，缺乏中間過渡
- **症狀**: 關係發展不自然，玩家無法代入
- **解決**: 設計明確的 Lv1-Lv4 過渡階段，每個階段有標誌性事件

### SE-3: 類型標籤化

- **嚴重度**: medium
- **情境**: 依賴類型設計角色
- **原因**: 只看表面特徵，忽略內在動機
- **症狀**: 角色變成「傲嬌機器」，缺乏深度
- **解決**: 先設計「為什麼是這個類型」的背景故事，再賦予類型特徵
