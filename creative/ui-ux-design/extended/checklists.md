# UI/UX 設計檢查清單

## WCAG POUR 無障礙檢查清單

### Perceivable（可感知）
- [ ] 所有圖片有 alt 文字
- [ ] 色彩不是唯一的資訊傳達方式
- [ ] 對比度符合 AA 標準（普通文字 4.5:1，大型文字 3:1）
- [ ] 影片有字幕
- [ ] 音訊有文字替代

### Operable（可操作）
- [ ] 所有功能可用鍵盤操作
- [ ] Focus 狀態明顯
- [ ] 沒有會導致癲癇的閃爍內容（每秒 < 3 次）
- [ ] 提供跳過重複內容的方式
- [ ] 頁面標題有意義

### Understandable（可理解）
- [ ] 語言明確指定（lang 屬性）
- [ ] 表單有清楚的標籤和錯誤提示
- [ ] 導航一致
- [ ] 錯誤訊息清楚說明問題和解法
- [ ] 重要操作可確認或撤銷

### Robust（健壯）
- [ ] 語意化 HTML
- [ ] ARIA 標籤正確使用
- [ ] 跨瀏覽器/輔助工具相容
- [ ] 程式碼通過 W3C 驗證

## 可用性測試準備清單

### 1. 定義目標
- [ ] 明確要驗證的假設
- [ ] 定義成功指標（任務完成率、時間、滿意度）

### 2. 設計任務
- [ ] 準備 3-5 個核心任務
- [ ] 任務模擬真實使用情境
- [ ] 任務描述不暗示答案

### 3. 招募受試者
- [ ] 招募 5-8 人（可發現 80% 問題）
- [ ] 受試者符合目標用戶特徵
- [ ] 準備適當酬勞

### 4. 準備環境
- [ ] 原型/測試版本就緒
- [ ] 錄影/錄音設備測試
- [ ] 觀察記錄表準備好

### 5. 測試引導話術
- 開場：「我們在測試產品，不是測試你。沒有對錯之分。」
- 過程：「請一邊操作一邊說出你的想法。」
- 卡住時：「你期望這裡會發生什麼？」

## 上線前檢查清單

### 導航
- [ ] 用戶知道自己在哪裡
- [ ] 用戶知道可以去哪裡
- [ ] 用戶可以輕鬆返回
- [ ] 麵包屑/導航顯示正確

### 內容
- [ ] 重要資訊在折疊線之上
- [ ] 文案清晰、無歧義
- [ ] 錯誤訊息有幫助
- [ ] 無錯字與斷句問題

### 互動
- [ ] 可點擊元素看起來可點擊
- [ ] 操作有回饋（視覺/聽覺）
- [ ] 表單驗證即時
- [ ] hover/focus/active 狀態完整

### 效能
- [ ] 首次載入 < 3 秒
- [ ] 互動回應 < 100ms
- [ ] 動畫流暢（60fps）
- [ ] 圖片已優化

### 跨裝置
- [ ] 手機版測試通過
- [ ] 平板版測試通過
- [ ] 主流瀏覽器測試通過
- [ ] 觸控操作正常

### 無障礙
- [ ] axe/WAVE 無嚴重問題
- [ ] 鍵盤導航順暢
- [ ] 螢幕閱讀器測試通過
