# 創意發想 Brainstorming & Ideation

> 系統化的創意思維方法，從靈感到可執行方案

## 適用場景

- 新產品/功能創意發想
- 解決方案腦力激盪
- 創意瓶頸突破
- 團隊創意工作坊
- 個人靈感激發

## 創意思維框架

```
┌─────────────────────────────────────────────────────────────────┐
│  創意思維流程                                                   │
│                                                                 │
│  發散思考 (Divergent) → 探索 → 收斂思考 (Convergent)           │
│                                                                 │
│  ┌─────────────┐     ┌─────────────┐     ┌─────────────┐       │

│  │ • 不評判    │     │ • 類比思考  │     │ • 優先排序  │       │

│  規則：先發散再收斂，不要同時進行！                             │

```

## 頭腦風暴方法

### 經典頭腦風暴

```markdown
## 頭腦風暴規則

1. **延遲評判** - 不批評任何想法
2. **數量優先** - 追求點子數量
3. **歡迎瘋狂** - 越大膽越好
4. **組合改進** - 在他人想法上延伸

## 執行流程

1. 明確定義問題（5分鐘）
2. 個人靜默發想（5分鐘）
3. 輪流分享不評論（15分鐘）
4. 自由聯想延伸（10分鐘）
5. 分類整理（5分鐘）
6. 投票選擇（5分鐘）
```

### SCAMPER 技法

| 字母 | 技法 | 提問 |
|------|------|------|
| **S** | Substitute 替代 | 可以替換什麼材料/人/過程？ |
| **C** | Combine 組合 | 可以和什麼結合？ |
| **A** | Adapt 改造 | 可以從哪裡借鑒？ |
| **M** | Modify 修改 | 可以放大/縮小/改變什麼？ |
| **P** | Put to other uses 他用 | 可以用在其他地方嗎？ |
| **E** | Eliminate 刪減 | 可以去掉什麼？ |
| **R** | Reverse 逆向 | 可以顛倒/重組嗎？ |

## 靈感激發技巧

### 隨機刺激法

```markdown
1. 隨機選擇一個詞（名詞最佳）
2. 列出這個詞的特徵
3. 強制連結到問題
4. 產生新想法

範例：
問題：如何提升用戶參與度？
隨機詞：樹
樹的特徵：生長、分支、季節變化、提供遮蔭
連結想法：
- 像樹一樣成長 → 用戶等級系統
- 分支 → 多元互動方式
- 季節變化 → 限時活動
- 遮蔭 → 互助社群
```

### 類比思考法

| 領域 | 類比問題 |
|------|----------|
| **自然界** | 大自然是如何解決這個問題的？ |
| **其他產業** | 酒店/航空/遊戲業是怎麼做的？ |
| **歷史** | 歷史上有類似的挑戰嗎？ |
| **藝術** | 藝術家會怎麼處理？ |

### 逆向思考法

```markdown
正向問題：如何讓用戶更滿意？
逆向問題：如何讓用戶最不滿意？

列出讓用戶不滿意的方法：
1. 回應慢
2. 介面複雜
3. 功能隱藏
4. 沒有幫助文檔

逆轉為解決方案：
1. 即時回應系統
2. 簡潔介面
3. 功能顯眼
4. 完善幫助中心
```

## 創意評估矩陣

### 可行性 vs 影響力

```
影響力
   ↑
高 │ ⭐ Quick Wins │ 🚀 Big Bets │
   │    優先執行   │   策略投資  │
   │───────────────┼─────────────│
低 │ 🗑️ 可捨棄    │ ❓ 需驗證   │
   │              │   原型測試  │
   └───────────────┴─────────────→
   低              高          可行性
```

### 評估標準

| 維度 | 評分項目 |
|------|----------|
| **新穎性** | 是否有獨特視角？ |
| **可行性** | 技術/資源可實現？ |
| **價值性** | 解決真實問題？ |
| **時效性** | 現在是好時機？ |

## 創意工作坊流程

```markdown
## 90 分鐘創意工作坊

### 暖身（10分鐘）
- 破冰活動
- 介紹規則

### 問題定義（10分鐘）
- HMW（How Might We）格式
- 範例：「我們如何能讓新用戶更快上手？」

### 發散階段（30分鐘）
- 個人發想（10分鐘）
- SCAMPER 延伸（10分鐘）
- 類比借鑒（10分鐘）

### 探索階段（15分鐘）
- 組合相似想法
- 延伸有潛力的

### 收斂階段（20分鐘）
- 點投票（每人 3 票）
- 討論前三名
- 選出執行方案

### 行動計劃（5分鐘）
- 下一步是什麼？
- 誰負責？
```

### 靈感來源

| 類型 | 來源 |
|------|------|
| **閱讀** | 書籍、文章、論文 |
| **觀察** | 日常生活、用戶行為 |
| **對話** | 同事、朋友、客戶 |
| **跨界** | 其他產業、藝術、科學 |
| **復盤** | 過去的成功/失敗經驗 |

## 突破創意瓶頸

```markdown
## 卡住時的急救包

1. **換個環境** - 離開桌子、出去走走
2. **換個視角** - 如果是新人會怎麼想？
3. **換個問題** - 重新定義問題
4. **換個約束** - 加入限制反而激發創意
5. **睡一覺** - 讓潛意識工作

## 常見阻礙

| 阻礙 | 解決 |
|------|------|
| 害怕被批評 | 延遲評判規則 |
| 追求完美 | 先求量再求質 |
| 思維固化 | 用隨機刺激打破 |
| 缺乏資訊 | 先做研究再發想 |
```
