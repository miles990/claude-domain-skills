# 創意發想 Brainstorming & Ideation

> 系統化的創意思維方法，從靈感到可執行方案

## 適用場景

- 新產品/功能創意發想
- 解決方案腦力激盪
- 創意瓶頸突破
- 團隊創意工作坊
- 個人靈感激發

## 創意思維框架

```
┌─────────────────────────────────────────────────────────────────┐
│  創意思維流程                                                   │
│                                                                 │
│  發散思考 (Divergent) → 探索 → 收斂思考 (Convergent)           │
│                                                                 │
│  ┌─────────────┐     ┌─────────────┐     ┌─────────────┐       │
│  │   發散      │     │   探索      │     │   收斂      │       │
│  │   ────      │     │   ────      │     │   ────      │       │
│  │ • 數量優先  │ ──→ │ • 連結組合  │ ──→ │ • 評估篩選  │       │
│  │ • 不評判    │     │ • 類比思考  │     │ • 優先排序  │       │
│  │ • 求異求新  │     │ • 逆向思考  │     │ • 可行分析  │       │
│  └─────────────┘     └─────────────┘     └─────────────┘       │
│                                                                 │
│  規則：先發散再收斂，不要同時進行！                             │
└─────────────────────────────────────────────────────────────────┘
```

## 頭腦風暴方法

### 經典頭腦風暴

```markdown
## 頭腦風暴規則

1. **延遲評判** - 不批評任何想法
2. **數量優先** - 追求點子數量
3. **歡迎瘋狂** - 越大膽越好
4. **組合改進** - 在他人想法上延伸

## 執行流程

1. 明確定義問題（5分鐘）
2. 個人靜默發想（5分鐘）
3. 輪流分享不評論（15分鐘）
4. 自由聯想延伸（10分鐘）
5. 分類整理（5分鐘）
6. 投票選擇（5分鐘）
```

### SCAMPER 技法

| 字母 | 技法 | 提問 |
|------|------|------|
| **S** | Substitute 替代 | 可以替換什麼材料/人/過程？ |
| **C** | Combine 組合 | 可以和什麼結合？ |
| **A** | Adapt 改造 | 可以從哪裡借鑒？ |
| **M** | Modify 修改 | 可以放大/縮小/改變什麼？ |
| **P** | Put to other uses 他用 | 可以用在其他地方嗎？ |
| **E** | Eliminate 刪減 | 可以去掉什麼？ |
| **R** | Reverse 逆向 | 可以顛倒/重組嗎？ |

```
┌─────────────────────────────────────────────────────────────────┐
│  Six Thinking Hats                                              │

│  🎩 白帽 - 事實數據    「我們知道什麼？」                       │

└─────────────────────────────────────────────────────────────────┘
```

## 靈感激發技巧

### 隨機刺激法

```markdown
1. 隨機選擇一個詞（名詞最佳）
2. 列出這個詞的特徵
3. 強制連結到問題
4. 產生新想法

範例：
問題：如何提升用戶參與度？
隨機詞：樹
樹的特徵：生長、分支、季節變化、提供遮蔭
連結想法：
- 像樹一樣成長 → 用戶等級系統
- 分支 → 多元互動方式
- 季節變化 → 限時活動
- 遮蔭 → 互助社群
```

### 類比思考法

| 領域 | 類比問題 |
|------|----------|
| **自然界** | 大自然是如何解決這個問題的？ |
| **其他產業** | 酒店/航空/遊戲業是怎麼做的？ |
| **歷史** | 歷史上有類似的挑戰嗎？ |
| **藝術** | 藝術家會怎麼處理？ |

### 逆向思考法

```markdown
正向問題：如何讓用戶更滿意？
逆向問題：如何讓用戶最不滿意？

列出讓用戶不滿意的方法：
1. 回應慢
2. 介面複雜
3. 功能隱藏
4. 沒有幫助文檔

逆轉為解決方案：
1. 即時回應系統
2. 簡潔介面
3. 功能顯眼
4. 完善幫助中心
```

## 創意評估矩陣

### 可行性 vs 影響力

```
影響力
   ↑
高 │ ⭐ Quick Wins │ 🚀 Big Bets │
   │    優先執行   │   策略投資  │
   │───────────────┼─────────────│
低 │ 🗑️ 可捨棄    │ ❓ 需驗證   │
   │              │   原型測試  │
   └───────────────┴─────────────→
   低              高          可行性
```

### 評估標準

| 維度 | 評分項目 |
|------|----------|
| **新穎性** | 是否有獨特視角？ |
| **可行性** | 技術/資源可實現？ |
| **價值性** | 解決真實問題？ |
| **時效性** | 現在是好時機？ |

## 創意工作坊流程

```markdown
## 90 分鐘創意工作坊

### 暖身（10分鐘）
- 破冰活動
- 介紹規則

### 問題定義（10分鐘）
- HMW（How Might We）格式
- 範例：「我們如何能讓新用戶更快上手？」

### 發散階段（30分鐘）
- 個人發想（10分鐘）
- SCAMPER 延伸（10分鐘）
- 類比借鑒（10分鐘）

### 探索階段（15分鐘）
- 組合相似想法
- 延伸有潛力的

### 收斂階段（20分鐘）
- 點投票（每人 3 票）
- 討論前三名
- 選出執行方案

### 行動計劃（5分鐘）
- 下一步是什麼？
- 誰負責？
```

| **閱讀** | 書籍、文章、論文 |
| **觀察** | 日常生活、用戶行為 |
| **對話** | 同事、朋友、客戶 |

## 突破創意瓶頸

```markdown
## 卡住時的急救包

1. **換個環境** - 離開桌子、出去走走
2. **換個視角** - 如果是新人會怎麼想？
3. **換個問題** - 重新定義問題
4. **換個約束** - 加入限制反而激發創意
5. **睡一覺** - 讓潛意識工作

## 常見阻礙

| 阻礙 | 解決 |
|------|------|
| 害怕被批評 | 延遲評判規則 |
| 追求完美 | 先求量再求質 |
| 思維固化 | 用隨機刺激打破 |
| 缺乏資訊 | 先做研究再發想 |
```

## 進階創意技法

### 設計思考 (Design Thinking)

```
┌─────────────────────────────────────────────────────────────────┐
│  Design Thinking 五階段                                          │
│                                                                 │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌────────┐│
│  │ Empathize│→│ Define  │→│ Ideate  │→│Prototype│→│  Test  ││
│  │  同理心  │  │  定義   │  │  發想   │  │  原型   │  │  測試  ││
│  └─────────┘  └─────────┘  └─────────┘  └─────────┘  └────────┘│
│                                                                 │
│  • Empathize: 觀察用戶、訪談、體驗                              │
│  • Define: 定義問題、找出洞察                                   │
│  • Ideate: 大量發想、不設限                                     │
│  • Prototype: 快速製作原型                                      │
│  • Test: 用戶測試、迭代改進                                     │
│                                                                 │
│  特點：非線性、可回退、以人為本                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 創意矩陣法 (Morphological Matrix)

```markdown
## 創意矩陣範例：設計新飲料

| 維度 | 選項 A | 選項 B | 選項 C | 選項 D |
|------|--------|--------|--------|--------|
| **基底** | 茶 | 咖啡 | 果汁 | 氣泡水 |
| **口味** | 甜 | 酸 | 苦 | 鹹 |
| **溫度** | 冰 | 常溫 | 熱 | 冰沙 |
| **容器** | 杯 | 瓶 | 罐 | 袋 |
| **場景** | 早餐 | 下午茶 | 運動後 | 睡前 |

隨機組合：咖啡 + 鹹 + 冰 + 袋 + 運動後
→ 鹹味冰咖啡運動飲料？

系統化生成：4×4×4×4×4 = 1024 種組合
```

### 奔馳法 (TRIZ)

```markdown
## TRIZ 40 個發明原理（精選）

| # | 原理 | 應用問題 |
|---|------|----------|
| 1 | 分割 | 如何讓大東西變靈活？ |
| 2 | 抽取 | 什麼部分可以獨立出來？ |
| 10 | 預先作用 | 如何提前準備？ |
| 13 | 逆向 | 反過來做會怎樣？ |
| 17 | 維度變化 | 從 2D 變 3D？平面到立體？ |
| 25 | 自服務 | 如何讓系統自我維護？ |
| 35 | 參數變化 | 改變物理狀態會怎樣？ |
| 40 | 複合材料 | 結合不同材料的優點？ |

## 矛盾矩陣
當改善 A 會惡化 B 時，查表找原理
例：增加強度 vs 減輕重量 → 複合材料、中空結構
```

## AI 輔助創意發想

```markdown
## AI 創意夥伴用法

### 角色扮演法
「假設你是一個 10 歲小孩，你會怎麼解決 [問題]？」
「假設你是 Steve Jobs，你會怎麼設計這個產品？」
「假設你是外星人第一次看到這個，你會有什麼疑問？」

### 強制連結法
「給我 5 個隨機名詞，然後強制連結到 [問題]」
「把 [問題] 和 [隨機領域] 結合，會產生什麼創意？」

### 約束創意法
「如果只能用 100 元解決這個問題，你會怎麼做？」
「如果只有 1 小時，你會怎麼做？」
「如果完全不能用科技，你會怎麼做？」

### 批判再創造
「這個想法的最大弱點是什麼？如何克服？」
「如果競爭對手要攻擊這個想法，他們會怎麼說？」
「把這個想法的相反做出來，會是什麼？」
```

## 創意熱身遊戲

### 30 Circles 練習

```markdown
## 30 個圓圈挑戰

準備：畫 30 個相同大小的空心圓
時間：3 分鐘
任務：把每個圓圈變成不同的東西

目的：
• 打破完美主義
• 訓練快速發想
• 熱身創意肌肉

評估：
• 完成數量（流暢性）
• 類別多樣性（靈活性）
• 獨特程度（原創性）
• 細節程度（精緻性）
```

### 「是的，而且...」練習

```markdown
## Yes, And... 即興練習

規則：
1. A 說一個想法
2. B 必須說「是的，而且...」然後延伸
3. 不能說「不」「但是」「可是」
4. 輪流進行 5 分鐘

範例：
A: 我們開一家咖啡店
B: 是的，而且咖啡店裡有貓
A: 是的，而且貓會幫忙端咖啡
B: 是的，而且客人可以領養貓...

目的：建立接納文化、練習延伸思維
```

## 創意會議主持技巧

```markdown
## 主持人 Checklist

### 會前準備
- [ ] 明確定義問題（HMW 格式）
- [ ] 準備便利貼和白板
- [ ] 邀請多元背景參與者
- [ ] 準備熱身活動
- [ ] 設定時間分配

### 會議中
- [ ] 說明規則（延遲評判、數量優先）
- [ ] 控制時間節奏
- [ ] 保護安靜的人發言
- [ ] 阻止過早批判
- [ ] 記錄所有想法

### 會後
- [ ] 整理分類想法
- [ ] 發送會議記錄
- [ ] 指派後續行動
- [ ] 追蹤進度

## 處理困難情況

| 情況 | 處理方式 |
|------|----------|
| 有人一直批評 | 提醒規則，記下批評稍後討論 |
| 場面太安靜 | 用輪流發言、個人靜默發想 |
| 離題太遠 | 溫和拉回，但記錄離題想法 |
| 一人獨占 | 「謝謝！讓我們聽聽其他人」 |
| 陷入僵局 | 換個技法，或休息 5 分鐘 |
```
