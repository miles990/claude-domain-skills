# 卡牌構築 Roguelike 設計 (Deckbuilder Roguelike)

> 類 Slay the Spire 的單人卡牌構築 Roguelike 遊戲設計框架

## 適用場景

- 設計類 Slay the Spire 的卡牌戰鬥系統
- 卡牌數值平衡與效果設計
- 能量/資源系統設計
- 遺物與卡牌協同效應設計
- Roguelike 進度與 Meta 進度設計

## 快速指令

```bash
/deckbuilder card [卡牌名稱]      # 設計單張卡牌
/deckbuilder character [名稱]     # 設計角色卡池
/deckbuilder relic [稀有度]       # 設計遺物
/deckbuilder enemy [類型]         # 設計敵人
/deckbuilder balance [卡牌列表]   # 數值平衡檢查
/deckbuilder synergy [主題]       # 設計協同效應組合
```

## 核心系統架構

### 類 Slay the Spire 遊戲的 DNA

```
┌─────────────────────────────────────────────────────────────────┐
│  Deckbuilder Roguelike 核心元素                                  │
│                                                                 │
│  ┌──────────────┐   ┌──────────────┐   ┌──────────────┐        │
│  │  能量系統    │ + │  卡牌構築    │ + │  Roguelike   │        │
│  │  (資源限制)  │   │  (成長選擇)  │   │  (程序生成)  │        │
│  └──────────────┘   └──────────────┘   └──────────────┘        │
│         ↓                  ↓                  ↓                │
│     決策張力           Build 多樣性        重玩價值            │
│                                                                 │
│  + ┌──────────────┐   ┌──────────────┐                         │
│    │  遺物系統    │ + │  敵人意圖    │                         │
│    │  (永久增益)  │   │  (資訊博弈)  │                         │
│    └──────────────┘   └──────────────┘                         │
│           ↓                  ↓                                 │
│       Synergy 深度      戰術規劃                                │
└─────────────────────────────────────────────────────────────────┘
```

### 為什麼這個公式成功？

| 元素 | 作用 | 設計目標 |
|------|------|----------|
| **能量限制** | 強迫取捨 | 每張牌都是決策 |
| **隨機獎勵** | 被動構築 | 無法預設最佳策略 |
| **敵人意圖** | 資訊博弈 | 有意義的防禦選擇 |
| **遺物系統** | 永久改變 | 每局都不同體驗 |
| **卡組精簡** | 稀釋懲罰 | 少即是多 |

## 1. 能量系統 (Energy System)

### 標準能量模型

```
┌─────────────────────────────────────────────────────────────────┐
│  能量系統設計                                                    │
│                                                                 │
│  基礎值：每回合獲得 3 能量（可透過遺物/卡牌增加）               │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  能量 = 決策預算                                         │   │
│  │                                                         │   │
│  │  3 能量 → 約 2-3 張牌的行動空間                         │   │
│  │  0 費牌 → 「免費」但效果較弱                            │   │
│  │  高費牌 → 強力但消耗整回合                              │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  能量成本分布建議：                                              │
│  │ 成本 │ 比例  │ 定位                │                        │
│  │──────│───────│────────────────────│                        │
│  │ 0費  │ 10-15%│ 輔助/條件觸發       │                        │
│  │ 1費  │ 40-50%│ 主力/基礎效果       │                        │
│  │ 2費  │ 25-30%│ 強力/複合效果       │                        │
│  │ 3費  │ 10-15%│ 終結技/大招         │                        │
│  │ X費  │ 5%    │ 可變/傾倒資源       │                        │
└─────────────────────────────────────────────────────────────────┘
```

### 能量操控卡設計

| 類型 | 效果模板 | 設計考量 |
|------|----------|----------|
| **能量生成** | 獲得 X 能量 | 高價值，需平衡 |
| **成本降低** | 下一張牌 -X 費 | 組合潛力 |
| **X 費牌** | 消耗所有能量，效果 ×X | 回合結算器 |
| **0 費條件** | 滿足條件時 0 費 | 構築獎勵 |

## 2. 卡牌類型系統

### 五大卡牌類型

```
┌─────────────────────────────────────────────────────────────────┐
│  卡牌類型                                                        │
│                                                                 │
│  🔴 攻擊牌 (Attack)                                              │
│  ├─ 造成傷害                                                    │
│  ├─ 可能附帶效果（虛弱、中毒）                                  │
│  └─ 部分牌對護甲/敵人有額外效果                                 │
│                                                                 │
│  🔵 技能牌 (Skill)                                               │
│  ├─ 獲得格擋 (Block)                                            │
│  ├─ 抽牌/能量操作                                               │
│  ├─ Buff/Debuff                                                 │
│  └─ 卡組操作（棄牌、消耗）                                      │
│                                                                 │
│  🟣 能力牌 (Power)                                               │
│  ├─ 永久效果（持續整場戰鬥）                                    │
│  ├─ 打出後不進入棄牌堆                                          │
│  └─ 通常成本較高                                                │
│                                                                 │
│  ⚫ 狀態牌 (Status)                                              │
│  ├─ 負面效果，塞入卡組                                          │
│  ├─ 通常無法打出                                                │
│  └─ 戰鬥結束後移除                                              │
│                                                                 │
│  💀 詛咒牌 (Curse)                                               │
│  ├─ 永久負面效果                                                │
│  ├─ 需要特殊方式移除                                            │
│  └─ 帶來額外挑戰/風險                                           │
└─────────────────────────────────────────────────────────────────┘
```

### 卡牌稀有度

| 稀有度 | 顏色 | 出現率 | 特徵 |
|--------|------|--------|------|
| **基礎** | 灰 | 起始牌 | 簡單直接 |
| **普通** | 灰 | 60% | 構築基石 |
| **罕見** | 藍 | 37% | 策略定義 |
| **稀有** | 金 | 3%* | Build 核心 |

> *稀有度保底：每抽到一張普通牌，稀有率 +1%

### 卡牌關鍵詞

```yaml
# 卡牌修飾詞
Exhaust: 使用後移除（本場戰鬥不再出現）
Ethereal: 回合結束時若在手牌則消耗
Innate: 戰鬥開始時必定在起始手牌
Retain: 回合結束時不棄牌，保留至下回合
Unplayable: 無法主動使用

# 效果關鍵詞
Block: 格擋，抵擋傷害（回合結束消失）
Vulnerable: 易傷，受到傷害 +50%
Weak: 虛弱，造成傷害 -25%
Frail: 脆弱，獲得格擋 -25%
Strength: 力量，攻擊傷害 +X
Dexterity: 敏捷，格擋值 +X
```

## 3. 戰鬥系統

### 回合結構

```
┌─────────────────────────────────────────────────────────────────┐
│  回合流程                                                        │
│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │  回合開始                                                  │ │
│  │  ├─ 獲得能量（基礎 3）                                    │ │
│  │  ├─ 抽牌（基礎 5 張）                                     │ │
│  │  ├─ 觸發「回合開始」效果                                  │ │
│  │  └─ 格擋歸零（除非有 Barricade）                          │ │
│  └───────────────────────────────────────────────────────────┘ │
│                         ↓                                       │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │  玩家行動（自由階段）                                      │ │
│  │  ├─ 打出任意數量卡牌（需足夠能量）                        │ │
│  │  ├─ 使用藥水                                              │ │
│  │  └─ 結束回合                                              │ │
│  └───────────────────────────────────────────────────────────┘ │
│                         ↓                                       │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │  回合結束                                                  │ │
│  │  ├─ 觸發「回合結束」效果                                  │ │
│  │  ├─ 未使用手牌進入棄牌堆（Ethereal 牌消耗）               │ │
│  │  └─ 能量清零（除非有 Ice Cream）                          │ │
│  └───────────────────────────────────────────────────────────┘ │
│                         ↓                                       │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │  敵人行動                                                  │ │
│  │  ├─ 依據「意圖」執行動作                                  │ │
│  │  ├─ 攻擊先扣格擋，再扣血量                                │ │
│  │  └─ 決定下回合意圖                                        │ │
│  └───────────────────────────────────────────────────────────┘ │
│                         ↓                                       │
│                    新回合開始                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 敵人意圖系統

```
┌─────────────────────────────────────────────────────────────────┐
│  意圖類型                                                        │
│                                                                 │
│  ⚔️ 攻擊 — 顯示傷害數值                                         │
│  🛡️ 防禦 — 獲得格擋                                             │
│  💪 增益 — 為自己/隊友上 Buff                                    │
│  💫 減益 — 對玩家施加 Debuff                                     │
│  ❓ 未知 — 不顯示具體行動                                        │
│  🔄 多重 — 攻擊 + 其他效果                                       │
│                                                                 │
│  設計原則：                                                      │
│  • 玩家需要資訊來做決策 → 顯示意圖                              │
│  • 但不能太簡單 → 不顯示多回合計畫                              │
│  • 精英/Boss 可有未知意圖增加挑戰                               │
└─────────────────────────────────────────────────────────────────┘
```

### 傷害計算

```python
# 基礎傷害公式
def calculate_damage(base_damage, attacker, defender):
    damage = base_damage

    # 攻擊者加成
    damage += attacker.strength
    if attacker.has_weak:
        damage *= 0.75  # 虛弱 -25%

    # 防禦者加成
    if defender.has_vulnerable:
        damage *= 1.5  # 易傷 +50%

    # 護甲處理
    blocked = min(damage, defender.block)
    defender.block -= blocked
    actual_damage = damage - blocked

    return max(0, int(actual_damage))

# 格擋計算
def calculate_block(base_block, character):
    block = base_block
    block += character.dexterity
    if character.has_frail:
        block *= 0.75  # 脆弱 -25%
    return max(0, int(block))
```

## 4. 遺物系統 (Relic System)

### 遺物設計原則

```
┌─────────────────────────────────────────────────────────────────┐
│  遺物設計哲學                                                    │
│                                                                 │
│  「遺物應該改變你的決策，而不只是加數值」                       │
│                                                                 │
│  ✅ 好的遺物設計：                                               │
│  ├─ 改變策略優先級（例：消耗流變強）                            │
│  ├─ 創造新的 Synergy（例：每回合抽 1 張）                       │
│  └─ 定義 Build 方向（例：0 費牌專精）                           │
│                                                                 │
│  ❌ 避免的設計：                                                 │
│  ├─ 純數值加成（+5% 傷害）→ 無感                                │
│  ├─ 過於複雜 → 難以評估價值                                     │
│  └─ 無條件強力 → 必選，無趣                                     │
└─────────────────────────────────────────────────────────────────┘
```

### 遺物稀有度

| 稀有度 | 來源 | 設計目標 |
|--------|------|----------|
| **起始** | 角色自帶 | 定義角色風格 |
| **普通** | 精英/商店 | 通用增益 |
| **罕見** | 精英/商店 | Build 定義 |
| **稀有** | Boss | 遊戲改變者 |
| **特殊** | 事件 | 風險/收益權衡 |
| **商店** | 限商店 | 便利性/修復 |
| **Boss** | 限 Boss | 極強但有代價 |

### 遺物模板

```yaml
# 遺物設計模板
relic:
  name: 遺物名稱
  rarity: common | uncommon | rare | boss | shop | event

  effect:
    trigger: 觸發時機
    # 常見觸發：戰鬥開始、回合開始、打出卡牌、受傷時、擊殺時
    action: 具體效果

  synergy:
    - 與什麼卡牌/策略有協同

  anti_synergy:
    - 與什麼衝突或無用

  flavor_text: 背景故事
```

### 遺物範例

```yaml
# 能量類遺物
Lantern:
  rarity: common
  effect: "戰鬥開始時，獲得 1 能量"
  synergy: [高費牌, 回合 1 爆發]

# 抽牌類遺物
Bag of Preparation:
  rarity: uncommon
  effect: "戰鬥開始時，額外抽 2 張牌"
  synergy: [Innate 牌, 起手組合技]

# Build 定義類遺物
Shuriken:
  rarity: uncommon
  effect: "每打出 3 張攻擊牌，獲得 1 力量"
  synergy: [低費攻擊, 多段攻擊]

# Boss 遺物（強力但有代價）
Runic Dome:
  rarity: boss
  effect: "每回合 +1 能量，但無法看到敵人意圖"
  trade_off: 能量 vs 資訊
```

```
┌─────────────────────────────────────────────────────────────────┐
```

## 6. 卡牌數值平衡

### 基礎數值框架

```
┌─────────────────────────────────────────────────────────────────┐
│  數值基準線（以 1 能量為單位）                                   │
│                                                                 │
│  1 能量 ≈                                                       │
│  ├─ 6-8 傷害（基礎攻擊）                                        │
│  ├─ 5-6 格擋（基礎防禦）                                        │
│  ├─ 1 張抽牌                                                    │
│  ├─ 1 能量生成（本回合）                                        │
│  ├─ 輕度 Debuff（1 回合）                                       │
│  └─ 小型持續效果                                                │
│                                                                 │
│  價值換算：                                                      │
│  • Exhaust = -0.5 能量價值（代價）                              │
│  • Ethereal = -0.3 能量價值（代價）                             │
│  • Innate = +0.3 能量價值（收益）                               │
│  • Retain = +0.2 能量價值（收益）                               │
│  • 升級 = +0.5~1.0 能量價值                                     │
└─────────────────────────────────────────────────────────────────┘
```

### 卡牌設計模板

```yaml
# 卡牌設計模板
card:
  name: 卡牌名稱
  type: Attack | Skill | Power
  rarity: Basic | Common | Uncommon | Rare
  cost: 0-3 或 X

  base_effect:
    description: 基礎效果描述
    damage: 傷害值（如適用）
    block: 格擋值（如適用）

  upgraded_effect:
    description: 升級後效果
    improvements: [傷害+3, 成本-1, 附加效果]

  keywords: [Exhaust, Ethereal, Innate, Retain]

  synergy_tags: [力量, 格擋, 消耗, 毒]

  design_intent: 這張牌在什麼 Build 中發揮作用
```

### 卡牌平衡檢查表

```markdown
## 卡牌平衡 Checklist

### 數值合理性
- [ ] 效果/成本比例符合基準線
- [ ] 升級提升約 0.5-1.0 能量價值
- [ ] 與同費用牌橫向對比合理

### Build 定位
- [ ] 明確屬於某個 Build 原型
- [ ] 與至少 2-3 張牌有 Synergy
- [ ] 不會單卡過強（無條件必選）

### 遊戲體驗
- [ ] 效果容易理解
- [ ] 決策有趣（不是無腦使用）
- [ ] 視覺反饋清晰

### 稀有度匹配
- [ ] 普通：簡單直接
- [ ] 罕見：有條件/複雜效果
- [ ] 稀有：Build 核心/遊戲改變者
```

## 7. 角色設計

### 角色差異化

```
┌─────────────────────────────────────────────────────────────────┐
│  角色設計框架                                                    │
│                                                                 │
│  每個角色需要：                                                  │
│  ├─ 獨特機制（專屬系統）                                        │
```

## 10. Sharp Edges（常見陷阱）

### SE-1: 卡組膨脹問題

```markdown
## 問題
玩家傾向於拿太多牌，導致：
- 關鍵牌抽不到
- Build 不聚焦
- 遊戲體驗變差

## 解決
✅ 設計卡組上限或懲罰機制
✅ 提供移除卡牌的機會
✅ 教育玩家「不拿也是選擇」
✅ 設計 Synergy 鼓勵精簡
```

### SE-2: 必選牌/遺物問題

```markdown
## 問題
某些牌/遺物過強，變成「無腦必選」：
- 減少構築多樣性
- 遊戲策略深度降低
- 平衡困難

## 解決
✅ 所有選項都有 Trade-off
✅ 強力牌需要 Build 配合
✅ 使用數據追蹤選擇率
✅ 選擇率 >80% 需要 Nerf
```

### SE-3: 防禦無聊問題

```markdown
## 問題
玩家覺得「打格擋牌很無聊」：
- 傷害帶來滿足感
- 防禦是「被動」行為

## 解決
✅ 防禦牌也要有爽感
✅ 設計反擊、格擋轉攻擊機制
✅ 格擋累積系統
✅ 過量格擋的獎勵
```

### SE-4: 運氣 vs 技術平衡

```markdown
## 問題
玩家抱怨「運氣決定勝負」：
- 抽牌順序不好
- 沒拿到關鍵牌
- 事件運氣差

## 解決
✅ 保證基礎可玩性（不靠 Synergy 也能通關）
✅ 提供「選擇」而非純隨機
✅ 稀有卡保底機制
✅ 多種 Build 路線可選
```

## 11. 數據平衡工具

### 關鍵指標追蹤

```yaml
# 需要追蹤的數據
analytics:

  card_metrics:
    pick_rate: 選擇率（offered 時選擇的比例）
    win_rate: 持有該牌的勝率
    play_rate: 每場戰鬥使用次數
    upgrade_priority: 升級優先度

  relic_metrics:
    pick_rate: 選擇率
    win_rate: 持有勝率
    synergy_with: 常見搭配

  balance_flags:
    - "選擇率 > 80% → 可能過強"
    - "選擇率 < 5% → 可能過弱或定位不明"
    - "勝率差異 > 10% → 需要調整"
```

### 平衡調整原則

```
┌─────────────────────────────────────────────────────────────────┐
│  平衡調整優先級                                                  │
│                                                                 │
│  1. 修復 Bug / 非預期互動 — 最高優先                            │
│  2. 削弱「必選」牌/遺物 — 高優先                                │
│  3. 加強「從不選」的選項 — 中優先                               │
│  4. 微調數值 — 低優先                                           │
│                                                                 │
│  調整幅度：                                                      │
│  • 小調：±10-15%（數值微調）                                    │
│  • 中調：±20-30%（明顯改變）                                    │
│  • 大改：重新設計效果（謹慎使用）                               │
│                                                                 │
│  避免：                                                          │
│  ✗ 一次改太多東西                                               │
│  ✗ 沒有數據支持的改動                                           │
│  ✗ 過度反應社群抱怨                                             │
└─────────────────────────────────────────────────────────────────┘
```

## 範例：設計一張新牌

```yaml
# 設計過程示範

## 1. 確定定位
target_build: 力量流
rarity: Uncommon
desired_role: Build 加速器

## 2. 初版設計
card_v1:
  name: 狂暴姿態
  cost: 1
  type: Skill
  effect: "獲得 2 力量。本回合受到傷害 +25%"

## 3. 數值檢驗
analysis:
  - 1 能量獲得 2 力量 → 略強於基準
  - 有代價（受傷 +25%）→ 平衡
  - 需要考慮累積效果 → 可能太強

## 4. 調整
card_v2:
  name: 狂暴姿態
  cost: 1
  type: Skill
  effect: "獲得 2 力量。獲得 1 易傷"
  upgrade: "獲得 3 力量"

## 5. Synergy 檢查
synergies:
  - 與「多段攻擊」有強 Synergy ✓
  - 與「格擋流」衝突（可接受）✓
  - 不會單卡過強 ✓

## 6. 最終確認
final_check:
  - [ ] 效果清晰易懂
  - [ ] 有明確 Build 定位
  - [ ] 數值平衡
  - [ ] 升級有意義
```
